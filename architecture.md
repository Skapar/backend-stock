# Описание проекта

Наш проект - это простой и понятный backend, написанный на Go. Он работает как API: получает запросы, обрабатывает их и отдаёт ответы. Никаких лишних сложностей - всё максимально прямолинейно. Запускаем мы его через Docker, поэтому проект работает одинаково на любой машине, и не нужно вручную ставить Go или настраивать окружение - просто запускаешь контейнер, и всё готово.



## Основная логика приложения

Сердце системы - это Go-приложение. Оно принимает HTTP-запросы, проверяет данные, выполняет нужную бизнес-логику и отдаёт результат в виде JSON. Приложение монолитное, то есть всё находится внутри одного сервиса: у нас нет микросервисов, очередей сообщений или каких-то отдельных фоновых воркеров. Проще говоря, есть один backend, и он делает всё, что нужно.



## Dockerfile

Dockerfile описывает, как собрать приложение: берётся наш Go-код, компилируется в бинарник, и в контейнере потом запускается уже готовая собранная программа. Никаких настроек на стороне пользователя - всё прячется внутрь Docker.



## Docker Compose

Управляет запуском всего окружения файл docker-compose.yml. Там описано, как запускать backend-сервис, какие ему нужны порты, какие переменные окружения, что монтировать и т.д. docker-compose создаёт для нашего приложения отдельную «песочницу», где сервис спокойно работает внутри контейнера и никак не конфликтует с системой пользователя.



## Makefile

В проекте есть Makefile - он облегчает жизнь при разработке. Вместо того чтобы постоянно писать длинные команды, можно просто выполнить make build, make run и т.д. Это сильно экономит время и делает работу аккуратнее.



## Зависимости Go

Зависимости Go фиксируются в go.mod и go.sum, благодаря чему проект всегда собирается стабильно - независимо от того, где ты его запускаешь.



## Архитектурная схема

Если посмотреть на архитектуру в целом, она такая: клиент отправляет запрос backend его принимает выполняет нужные действия отправляет обратно JSON-ответ. Всё. Никаких внешних сервисов, никакой базы данных - текущая версия проекта специально сделана простой, чтобы показать сам принцип работы API. Но если понадобится, базу можно подключить в любой момент, потому что Docker уже есть, и к этому всё подготовлено.



## Итог

По итогу у нас выходит аккуратный монолитный Go-сервис, который запускается через Docker и работает одинаково на любой машине. На защите это очень удобно: просто запускаешь docker compose up, и проект тут же поднимается, готовый к тестированию.
