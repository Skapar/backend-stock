# backend


## Ответы на вопросы
```
1️.Как выбор структуры базы данных (SQL или NoSQL) влияет на дизайн CRUD API?

SQL (PostgreSQL, MySQL) требует строгой схемы таблиц, типов данных и связей. CRUD API обычно отражает структуру таблиц, и запросы выполняются через SQL (SELECT, UPDATE, INSERT, DELETE).

NoSQL (MongoDB) гибче — структура может меняться динамически, что подходит для документов, JSON-данных или быстро меняющихся схем.

Следовательно, SQL → строгие схемы и валидация, NoSQL → гибкость, но меньше гарантий целостности данных.

2️.Какие проблемы могут возникнуть при массовых обновлениях данных через API?

Нагрузка на базу (большие UPDATE могут блокировать таблицы).

Потеря данных при параллельных запросах (race condition).

Проблемы консистентности (частично обновлённые данные при сбое).

Сложность отката (rollback).

3️.Почему важно использовать правильные HTTP-методы (GET, POST, PUT, DELETE)?

GET — только чтение, безопасен и кэшируется.

POST — создание ресурса.

PUT / PATCH — изменение ресурса.

DELETE — удаление.
Использование только POST ломает семантику REST и мешает кэшированию, тестированию и читаемости API.
```

```
1️.Какие уязвимости могут возникнуть при хранении JWT на клиентской стороне?

Если хранить токен в localStorage — его могут украсть через XSS.

Если хранить в cookie без HttpOnly — доступен JavaScript-у.

Решение: хранить токен в HttpOnly cookies, использовать HTTPS и короткий срок жизни.

2️.В каких случаях стоит ограничивать время жизни JWT, и какие проблемы это создаёт для UX?

Ограничивать TTL нужно, чтобы уменьшить риск кражи токена.

Проблема: пользователю нужно будет повторно логиниться или использовать refresh-токен для продления сессии.

Компромисс: короткий access_token + долгий refresh_token.

3️.Как логирование помогает в расследовании инцидентов безопасности?

Позволяет отследить:

кто и когда вызывал API;

какие ошибки и статусы возвращались;

с каких IP выполнялись подозрительные действия.

Это помогает восстановить цепочку событий при атаке, определить источник взлома и улучшить защиту.
```
