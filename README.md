# backend


Инструкция по запуску 

Чтобы запустить проект, ничего сложного делать не нужно — всё работает через Docker. Главное, чтобы Docker и Docker Compose были установлены. Проверить, что они стоят и работают, можно обычными командами:

docker --version
docker compose version


Если версии выводятся - значит всё норм, можно продолжать.

После этого просто клонируем репозиторий к себе:

git clone <https://github.com/Skapar/backend-stock>
cd backend-stock-main


Создаёшь файл .env рядом с docker-compose.yml и прописываешь туда нужные переменные. Например:

APP_PORT=8080


Теперь самое важное - запуск. Он делается одной командой через Docker Compose:

docker compose up --build


После этого Docker сам соберёт образ, поднимет контейнер и запустит backend.
Когда всё соберётся, сервис станет доступен по адресу, который прописан в docker-compose.yml. Чаще всего это:

http://localhost:8080


Если хочешь остановить проект - делаешь:

docker compose down


Если вдруг нужно запустить проект без Docker (например, для разработки), то:

Ставишь Go нужной версии (смотри в go.mod).

В корне проекта выполняешь:

go mod download
go run ./...


И Go сам поднимает сервис на своём порту.
